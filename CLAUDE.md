# CaseLaw — Project Instructions

## What This Project Is

CaseLaw is a SaaS legal-tech web application. This repository contains the **design system documentation** — design tokens, component specs, flow analyses, and Angular component implementations using **spartan.ng**. It is a designer-led project; the output is handoff documentation for the front-end engineering team.

## Critical Constraints

- **No business logic.** Components must be pixel-perfect but carry NO event handlers, data bindings, API calls, or state management. Output is static, visual-only component specs.
- **No Figma MCP.** All design extraction is done from screenshots in `docs/screenshots/`. Reference those images directly.
- **Confidence levels required.** Every token extraction must be tagged: `Exact`, `Estimated`, or `Needs verification`.

## Tech Stack

- Angular 20 (standalone components)
- spartan.ng (brain `0.0.1-alpha.644` + locally-owned helm directives)
- Tailwind CSS v4 (CSS-first `@theme` blocks, no `tailwind.config.js`)
- pnpm (package manager)
- PostCSS (`@tailwindcss/postcss`)
- Node 22+ (managed via `fnm`)

## Styling Rules

All colors flow through **spartan's CSS variable system** (`--background`, `--foreground`, `--primary`, etc.) defined in `src/styles.css`. Use Tailwind v4 utility classes that reference these variables:

```html
<!-- Correct: spartan semantic color tokens -->
<p class="text-sm font-semibold text-muted-foreground">Label</p>
<button hlmBtn variant="outline">Action</button>

<!-- Wrong: hardcoded neutral values -->
<p class="text-sm font-semibold text-neutral-700">Label</p>
```

## spartan.ng Import Patterns

Helm components are locally owned (generated by `ng g @spartan-ng/cli:ui`). Import via tsconfig path aliases:

```typescript
import { HlmButton } from '@spartan-ng/helm/button';
import { HlmInput } from '@spartan-ng/helm/input';
import { HlmSwitch } from '@spartan-ng/helm/switch';
import { HlmBadge } from '@spartan-ng/helm/badge';
// etc.
```

Brain primitives come from the npm package:
```typescript
import { BrnSelect } from '@spartan-ng/brain/select';
import { BrnDialog } from '@spartan-ng/brain/dialog';
```

## Design Tokens — Quick Reference

### Colors (OKLCH via spartan CSS variables)
- `--background` / `bg-background` — page bg (`oklch(0.985 0 0)` ≈ neutral-50)
- `--foreground` / `text-foreground` — primary text (`oklch(0.145 0 0)` ≈ neutral-950)
- `--card` / `bg-card` — card bg (`oklch(1 0 0)` = white)
- `--primary` / `bg-primary` — dark button bg (`oklch(0.145 0 0)`)
- `--muted-foreground` / `text-muted-foreground` — secondary text (`oklch(0.525 0 0)`)
- `--border` / `border-border` — borders (`oklch(0.922 0 0)`)
- `--success` / `bg-success` — green toggle/badge (`oklch(0.6 0.2 145)`)
- `--destructive` / `bg-destructive` — error red (`oklch(0.577 0.245 27.325)`)

### Typography
- Scale: `text-xs` (12px) → `text-3xl` (30px)
- Families: system sans-serif (UI default), `font-kalice` (Kalice Trial Medium, serif display, headings/wordmark)
- Weights: `font-normal` (400), `font-medium` (500), `font-semibold` (600), `font-bold` (700)

### Spacing & Radius
- Base radius: `--radius: 0.5rem` (spartan variable)
- Cards: `rounded-xl`, Inputs/buttons: `rounded-md`
- Shadows: `shadow-xs`, `shadow-sm`, `shadow-md`, `shadow-lg`

## Running the Project

```bash

pnpm start          # or: npx ng serve
```

Navigate to `http://localhost:4200` for the Design System showcase (landing page). Key routes:
- `/` — Design System showcase (tokens, components, engineering handoff notes)
- `/screens` — Screen index with links to all onboarding screens
- `/law-firm/0-type-selection` through `/law-firm/8-choose-plan`
- `/student/0-type-selection` through `/student/7-choose-plan`

## Workflow

1. Work **flow by flow** (onboarding is implemented; next flows TBD).
2. Analyze screenshots → extract tokens → document components → implement in Angular.
3. Tag all extractions with confidence levels.
4. Accumulate product context in `docs/product/product-context.md`.

## Resolved Decisions

- `type-selection-card` selected border: `ring-2 ring-ring` (avoids layout shift)
- Toggle active color: `bg-success` via class override `data-[state=checked]:bg-success`
- `date-select`: separate component with calendar popover (not a `select` variant)
- Plan prices/quotas: placeholder values pending product confirmation
- Modal screens: plain overlay backdrop (`bg-black/50`), no fake dashboard
- Modal screens use `modal-shell` component (static design spec); production should migrate to `HlmDialog` + `BrnDialog`
- Design System showcase is the landing page at `/`; old screen index moved to `/screens`
