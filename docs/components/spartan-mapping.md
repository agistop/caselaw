# Component Mapping — spartan.ng

Maps each CaseLaw UI need to its spartan.ng implementation. Components fall into two categories:

1. **Spartan helm** — generated by `ng g @spartan-ng/cli:ui`, locally owned in `src/app/components/helm/`
2. **Custom compositions** — built from spartan primitives, in `src/app/components/custom/`

---

## Spartan Helm Components (CLI-generated)

| CaseLaw Need | Spartan Directive/Component | Import Path | Customization |
|-------------|---------------------------|-------------|---------------|
| Dark/outline/link buttons | `hlmBtn` directive | `@spartan-ng/helm/button` | Variants via `variant` input: `default`, `outline`, `link`, `ghost` |
| Form text inputs | `hlmInput` directive | `@spartan-ng/helm/input` | Applied to `<input>` elements; styling via CSS variables |
| Form field labels | `hlmLabel` directive | `@spartan-ng/helm/label` | Applied to `<label>` elements |
| Persona/country/currency dropdowns | `brn-select` + `hlm-select-*` | `@spartan-ng/helm/select` + `@spartan-ng/brain/select` | Brain handles behavior, helm handles styling |
| Toggle switches | `hlm-switch` | `@spartan-ng/helm/switch` + `@spartan-ng/brain/switch` | Active color overridden with `data-[state=checked]:bg-success` |
| "Most Popular" pill | `hlmBadge` directive | `@spartan-ng/helm/badge` | Green variant via `bg-success text-success-foreground` |
| Type-selection/plan cards | `hlmCard*` directives | `@spartan-ng/helm/card` | `hlmCard`, `hlmCardHeader`, `hlmCardContent`, `hlmCardFooter` |
| Modal overlays | `hlm-dialog-*` | `@spartan-ng/helm/dialog` + `@spartan-ng/brain/dialog` | Firm details, collaborators, terms modals |
| Section dividers | `hlm-separator` | `@spartan-ng/helm/separator` | Default horizontal line |
| Step progress bar | `hlm-progress` | `@spartan-ng/helm/progress` | Not used directly — custom `progress-steps` component instead |
| Lucide icons | `hlm-icon` | `@spartan-ng/helm/icon` | Wraps `@ng-icons/lucide` icons |
| Scrollable ToS text | `hlm-scroll-area` | `@spartan-ng/helm/scroll-area` | Used in terms-privacy screen |

## Engineering Handoff Notes

### ModalShell → HlmDialog migration

`modal-shell` is a **static design-spec component** — it renders the modal permanently "open" with no open/close state, focus trapping, or keyboard handling. When implementing the real flow, replace it with spartan's dialog system:

| Design Spec (current) | Production Implementation |
|---|---|
| `<app-modal-shell>` | `<hlm-dialog>` + `<hlm-dialog-content>` |
| `<div class="fixed inset-0 bg-black/50">` backdrop | `<hlm-dialog-overlay>` (automatic via `BrnDialog`) |
| `<ng-content />` body slot | Content inside `<hlm-dialog-content>` |
| `<ng-content select="[footer]" />` | `<hlm-dialog-footer>` |
| Heading `<h2>` | `<hlm-dialog-header>` + `<hlm-dialog-title>` |
| `[currentStep]` / `[totalSteps]` inputs | Same — embed `<app-progress-steps>` inside dialog content |

**Imports needed:**
- Brain: `BrnDialog`, `BrnDialogTrigger`, `BrnDialogRef` from `@spartan-ng/brain/dialog`
- Helm: `HlmDialog`, `HlmDialogContent`, `HlmDialogOverlay`, `HlmDialogHeader`, `HlmDialogTitle`, `HlmDialogFooter` from `@spartan-ng/helm/dialog`

The helm dialog components are already scaffolded at `src/app/components/helm/dialog/`.

### Select dropdowns

The modal screens use hand-rolled select divs (styled `<div>` with chevron icon) rather than `HlmSelect` + `BrnSelect`. This is intentional for the design spec — `BrnSelect` requires functional wiring (open/close, option selection, value binding). When implementing, replace the static select divs with the full `brn-select` + `hlm-select-*` component set from `@spartan-ng/brain/select` + `@spartan-ng/helm/select`.

## Routing

All screens are lazy-loaded via `src/app/app.routes.ts`.

| Route Pattern | Example | Description |
|---------------|---------|-------------|
| `/` | `http://localhost:4200/` | Design System showcase (tokens, components, engineering handoff notes) |
| `/screens` | `http://localhost:4200/screens` | Screen index with links to all onboarding screens |
| `/law-firm/{N}-{name}` | `/law-firm/3-create-account` | Law Firm onboarding flow screens |
| `/student/{N}-{name}` | `/student/2-create-account` | Student onboarding flow screens |
